To first make a website you need to have a reliable coding site. Sites like Github and Unity are good coding sites however there are certain sites better for websites or games. Those are sites you can download and use for free but there are some core features on a laptop or PC that can be used to code such as Terminal. When I was first coding websites I had orignially started using Terminal but found Github to be more practical. Choosing a coding site depends on what exactly you're coding and if the coding site itself suits your specfic taste (or if you're like me need to use a specfic site for a good grade) deciding on a coding site entirely depends on you. For this specific project we were given a base code to build off of, the code itself consisted of 2 major componenets the body and the head. The head held all the background code that wouldn't be visible on the actual website and affect the very title of the website in the tab bar and the icon sitting next to it as well as other aspects. To fully view the head code you'd need to inspect the actual website you see before you. Now the head code itself is called this for the main reason of being the shortest and top of the code, essentaily topping or being the head of the main body code. The body code itself is the real "meat" of the website holding nearly all of the actual code and its visuals. The visuals you see now are apart of the body's code that I had programed, including the layout, links, images, videos, headings, and this very paragraph! But the body code doesn't do all the visuals, that's what the style.css is for, all of the colors, spacing, and fonts are apart of Style.css and affect all websites its linked to. Which you may have noticed if you checked on the other websites before finishing reading all of this (shame on you...) that all of the websites have all of the same coloring, borders, fonts, etcera. That's all thanks to the Style.css which controls all of that.
